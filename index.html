Security Case Study ‚Äî AWS Threat Detection Lab

Documented to meet TKH portfolio rubric: Executive Summary, Goal, Methodology, Findings, Remediation.

Executive Summary

Built a small, cloud-based blue-team lab in AWS to detect simulated threats using GuardDuty and CloudTrail. The goal was to demonstrate understanding of cloud visibility, alerting, and remediation ‚Äî without using production resources.

Goal & Scope

  • Simulate suspicious activity (unusual API calls, privilege escalation).
  • Verify logging was centralized (CloudTrail).
  • Route alerts to a SOC-style view using Security Hub.

Findings

  • Detected misconfigured IAM permissions that allowed broader access than intended.
  • Observed that without GuardDuty enabled, some events would have gone unnoticed.

Remediation & Risk

  • Enabled GuardDuty + Security Hub across the account.
  • Reduced IAM policy from broad `*` actions to least-privilege.
  • Documented steps in GitHub (README) so the lab can be re-run by another learner.
