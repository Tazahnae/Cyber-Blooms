Case Study ‚Äî Kali ‚Üí Metasploitable 2 via pfSense

Author: TaZahnae Matthews ‚Ä¢ Platform: UTM (M1 Mac) ‚Ä¢ Scope: Kali (attacker), Metasploitable 2 (target), pfSense (firewall)

Objective

I have Built a fully isolated virtual range to simulate a cyber-attack that routes traffic from Kali Linux (attacker) to Metasploitable 2 (target) through pfSense. Demonstrate enumeration ‚Üí exploitation ‚Üí monitoring ‚Üí hardening in a safe, repeatable environment.

Methodology

  • Provisioned VMs in UTM with distinct WAN/LAN segments and hardened base images.
  • Performed reconnaissance using Nmap and mapped exploitable services.
  • Used Metasploit Framework to validate vulnerabilities selected for the exercise.
  • Tuned pfSense firewall rules and verified that logs captured attacker actions.
  • Documented a runbook for setup, teardown, and safe evidence capture.

Tools Used

  • Kali Linux, Metasploitable 2, pfSense
  • Nmap (recon), Metasploit (validation), Linux CLI utilities
  • UTM on macOS (Apple M-series) for isolation & networking

Findings

  • Metasploitable 2 exposed several exploitable, intentionally vulnerable services.
  • Default firewall/open rules allowed more permissive traffic than necessary; tightening rules reduced exposure.
  • Segmentation plus logging produced clear, actionable telemetry for response.

Remediation

  • Hardened pfSense with least-privilege rules for required service ports.
  • Disabled unused interfaces and restricted management access to trusted hosts.
  • Captured runbook and validated re-deploy steps for reproducibility.

Outcomes

  • A safe, reusable attacker/defender lab that demonstrates the full security lifecycle.
  • Concrete evidence and documentation suitable for training or portfolio review.
  • Next steps: integrate IDS/IPS and SIEM for richer telemetry and alerting exercises.

Lessons Learned

  • Firewall defaults can overexpose services‚Äîdeny-by-default with explicit allows is safer.
  • Repeatable runbooks reduce setup mistakes and speed verification.
  • Segmentation + logging provide the clearest forensic story during attacks.
